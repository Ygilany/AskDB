AskDB Architecture

Natural language → schema-grounded SQL. A layered monorepo of reusable contracts, database integrations, and first-party surfaces.

System Overview The central runtime boundary and high-level data path

Core Principle

AskDB returns SQL for review. The host application owns approval, execution, database roles, tenant policy, network controls, and audit logging. @askdb/core never executes generated SQL.

Reusable Contracts & Libraries

@askdb/core
v0.5.0-beta · packages/core
The dialect-agnostic NL→SQL pipeline engine. Provides ask() orchestration, Schema v2 loading/parsing, enrichment suggestions, operating modes, Pino structured logging, and retrieval input. Central contract for all other packages.
library contract
No database drivers · No generated-SQL execution · No engine-specific connector
@askdb/introspect
packages/introspect
Engine-agnostic Connector<TInput> contract, introspection orchestrator, and Schema v2 renderer. Defines the intermediate SQL IR (SqlSchema, SqlTable, SqlColumn, SqlForeignKey, …) that every connector produces.
library contract
No default connector · No engine-specific input union · No standalone binary
@askdb/rag
packages/rag
Deterministic Schema v2 chunking (table, column, cql, question, concept, relationship chunks), BYO Embedder / VectorStore / Retriever interfaces, and built-in adapters: in-memory, file-backed, pgvector. All stores (createMemoryStore, createFileStore, createPgvectorStore) and embedders (createAiSdkEmbedder, createOpenAiEmbedder) are exported from the root barrel — no sub-path imports required. Also ships a setup-store CLI command for pgvector schema provisioning.
rag library
Optional layer — core SQL generation still flows through @askdb/core + dialect
@askdb/enrich
packages/enrich
Headless Schema v2 authoring helpers: loadWorkspace(), table draft builders, saveTable(), concepts editing, markdown preservation with frontmatter, suggestion context builders, and bundleSchemaDirectory(). Shared logic for TUI and Studio.
library headless
No terminal or browser UI · No introspection · No database connection

Integration Packages (Database Connectors)

@askdb/postgres
packages/postgres
Postgres dialect (postgresDialect), createPostgresConnector() with live and from-export modes, catalog SQL templates (POSTGRES_TEMPLATE_BUNDLE), and optional pg-backed catalog runner.
connector dialect
pg is optional peer — only for live introspection. Generated SQL still executes outside AskDB.
@askdb/prisma
packages/prisma
Offline Prisma schema-file connector. Reads one schema.prisma or a directory of .prisma files via @prisma/internals and renders Schema v2 physical metadata. Auto-discovers the Prisma project root.
connector offline
No live database connection · No SQL dialect · Pair output with a dialect for SQL generation
@askdb/mysql
packages/mysql
MySQL/MariaDB integration. Exports MYSQL_DIALECT, MARIADB_DIALECT, createMysqlConnector(), and a mysql2-backed catalog runner. NL→SQL prompt and validator live in @askdb/core.
connector dialect
Introspection connector only · Dialect spec lives in @askdb/core
@askdb/sqlite
packages/sqlite
SQLite integration. Exports SQLITE_DIALECT, createSqliteConnector(), and a better-sqlite3-backed catalog runner for reading SQLite database files.
connector dialect
Introspection connector only · Dialect spec lives in @askdb/core
@askdb/sqlserver
packages/sqlserver
Microsoft SQL Server (T-SQL) integration. Exports SQLSERVER_DIALECT, createSqlServerConnector(), and an mssql-backed catalog runner.

Introspection connector only · Dialect spec lives in @askdb/core

First-party Product Surfaces (apps/)

askdb (CLI)
apps/cli · bin: askdb
Batteries-included askdb binary. Commands: ask (NL→SQL), introspect (schema discovery — engine and connection URL now resolved from askdb.config.ts without flags), enrich (launch TUI), studio (launch browser UI), init (bootstrap config), bundle (pack schema directory). Composed from all reusable packages.
CLI batteries-included
Product surface that composes packages · Not the reusable contract layer
@askdb/http-api
apps/http-api · bin: askdb-http
Minimal HTTP wrapper around @askdb/core. Exposes POST /ask → SQL, GET /health. Accepts inline schemaJson or resolves from ASKDB_SCHEMA_PATH. Supports x-correlation-id header, request-body mode, and explain.
HTTP REST
No duplicate NL→SQL implementation · No SQL execution
@askdb/studio
apps/studio · bin: askdb-studio
Local browser UI serving a React SPA. Routes: /api/workspace, /api/tables/:id, /api/suggest, /api/rag/status, /api/rag/index, /api/rag/query, /api/ask. Integrates enrichment, RAG exploration, and sample NL→SQL with token usage tracking. RAG vector store is configurable via askdb.config.ts rag.store field: memory, file (default), or pgvector (auto-provisions schema via ensureSchema()). Default schema directory resolved from config without requiring a CLI flag.
browser UI React
Authoring surface over Schema v2 · Not a connector package
@askdb/tui
packages/tui · bin: askdb-tui
Maintained terminal enrichment surface built on Ink + React. Provides interactive table/column editing, concept authoring, AI suggestion review (when OPENAI_API_KEY is set), and schema bundling. Operates on Schema v2 files only.
TUI Ink/React
Operates on files · Does not introspect or connect to databases
@askdb/docs-site
apps/docs-site · Astro + Starlight
Static documentation site built with Astro/Starlight. Mirrors MDX content from the docs directory, providing searchable guides for architecture, connectors, RAG, Studio, TUI, and the HTTP API.
static site Astro
Markdown files remain canonical for architecture and contracts

Data Flows The four primary runtime flows through the system

1 · NL→SQL Pipeline (ask())

NL→SQL pipeline through ask(). RAG is an optional pre-step that replaces the full DDL with focused chunks.

2 · Introspection Flow

Introspection: connectors read from any source (live DB, export bundle, schema file) and emit a uniform SQL IR that renders into Schema v2.

3 · Enrichment Flow

Enrichment adds the describable layer on top of the physical schema. TUI and Studio share workspace logic via @askdb/enrich.

4 · RAG Indexing & Retrieval Flow

RAG indexes Schema v2 chunks once (skipping sensitive content by default). The retriever focuses the DDL block in ask() for large schemas, improving prompt quality and reducing token usage.

Schema v2 Format The describable-schema artifact — the single source of truth for NL→SQL context

On-disk Layout

my-app.schema/
  schema.json           # physical layer
  tables/
    users.md            # describable layer
    orders.md
    order_items.md
  concepts.md           # optional vocabulary
  schema.lock.json      # RAG content hashes
  schema.embeddings.json  # RAG vectors (JSON)
  schema.embeddings.bin   # RAG vectors (binary)

schema.json Structure

{
  "version": 2,
  "schemaId": "my-app",
  "provider": "postgres",  // optional
  "tables": [{
    "id": "table:public.users",
    "name": "users",
    "schema": "public",
    "sensitive": false,
    "columns": [{
      "id": "table:public.users#email",
      "name": "email",
      "type": "text",
      "nullable": false,
      "sensitive": true   // PII flag
    }]
  }]
}

RAG Chunk Types

table
Table name, description, aliases, primary entity
column
Column name, type, description, aliases
cql
Common query language snippets per table
question
Example questions from table markdown
concept
Cross-table domain vocabulary from concepts.md
relationship
Foreign key relationships between tables

Extension Points BYO interfaces and seams for custom integrations

Extension Point Interface Defined In When to Implement
SQL Dialect AskDialect or DialectSpec @askdb/core Add a database dialect or custom SQL generation policy. Built-in IDs: postgres, mysql, mariadb, sqlite, sqlserver, cockroachdb
Introspection Connector Connector<TInput> @askdb/introspect Add a new schema metadata source (new DB engine, export format, schema file type)
Catalog Runner CatalogQueryRunner @askdb/introspect Use a different live database client while keeping the connector unchanged
RAG Embedder Embedder = (texts: string[]) => Promise<number[][]> @askdb/rag Use a different embedding provider. Built-in helpers: AI SDK, OpenAI
Vector Store VectorStore (upsert/query/delete) @askdb/rag Use a different vector database. Built-in: in-memory, file-backed, pgvector
Retriever Retriever = (params) => Promise<QueryResult[]> @askdb/rag / @askdb/core Custom retrieval policy, hybrid search, or filtering logic
Enrichment UI Workspace, draft builders, save helpers @askdb/enrich Build a custom authoring UI without depending on TUI or Studio
Product Surface Compose packages into a workflow App packages or consumer app CLI, HTTP server, MCP server, browser SDK, Slack bot — host owns auth, policy, execution
