<p className="doc-eyebrow">Artifact contract</p>

<p className="doc-lede">An <strong>AskDB schema</strong> is a folder (or single bundle file) that pairs machine-readable structure with optional human-written context: one physical JSON layer, optional markdown per table, optional concepts, and bundle output when you need one portable file.</p>

<p>
  On disk, <code>{"schema.json"}</code> uses <code>{"\"version\": 2"}</code> to identify this layout. For field-level validation and bundling rules maintained with the source tree, see the{" "}
  <a href="https://github.com/Ygilany/AskDB/blob/main/docs/contracts/schema-v2.md">schema contract on GitHub</a>.
</p>

## Layout



```
my-app.schema/
  schema.json
  tables/
    users.md
    orders.md
  concepts.md
  schema.lock.json
```


          <p>Only <code>{"schema.json"}</code> is required. A directory with no table markdown files is valid.</p>

## Physical layer

<p><code>{"schema.json"}</code> stores stable table and column IDs, physical names, data types, nullability, primary keys, relationships, and sensitivity flags.</p>


```
{
  "version": 2,
  "schemaId": "orders-users",
  "tables": [
    {
      "id": "table:users",
      "name": "users",
      "columns": [
        {
          "id": "table:users#email",
          "name": "email",
          "type": "text",
          "nullable": false,
          "sensitive": true
        }
      ]
    }
  ]
}
```



## Enrichment layer (table docs)

<p><code>{"tables/<name>.md"}</code> adds aliases, descriptions, examples, common query language, and business context without changing physical metadata.</p>


```
---
id: table:orders
name: orders
schemaId: my-app
aliases: [purchases, sales]
columns:
  - id: table:orders#status
    enum: [pending, paid, shipped, cancelled]
    description: Order lifecycle state.
---

# Table: orders

Customer purchase orders. One row per submitted order.

## Common query language

- "sales" usually means paid orders
```



## Enrichment workflow

<p>The typical flow is introspect, enrich, re-introspect as the database changes, then bundle when another service needs one file.</p>


```
askdb introspect --url "$DATABASE_URL" --out my-app.schema --schema-id my-app
askdb enrich --schema my-app.schema
askdb bundle my-app.schema --out my-app.schema.bundle.json
```
