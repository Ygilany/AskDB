## Why AskDB

<div class="home-north-card">
  <p class="doc-lede home-north-lede">
    We optimize for <strong>trust-first analytics</strong>: the model sees your <strong>AskDB schema</strong> (structure plus optional table docs you approve)—not a dump of production data—and row-level fields stay sensitive by default. We optimize for a <strong>developer-first embed</strong>: install from npm, bring your own model keys, database connectivity, embeddings, and vector store when you need them. For philosophy and limitations, see the{" "}
    <a href="https://github.com/Ygilany/AskDB">AskDB project on GitHub</a>.
  </p>
</div>

## First run

<p class="home-terminal-intro">Install the CLI, create a schema folder, then ask a question against it.</p>

```bash
npm install askdb
npx askdb init
# add API keys to .env (see comments from init)

npx askdb introspect \
  --url "$DATABASE_URL" \
  --out my-app.schema \
  --schema-id my-app

npx askdb enrich --schema my-app.schema

npx askdb ask \
  --schema my-app.schema \
  --question "Which tables look active?"
```
