## North star

<p>
  We optimize for <strong>trust-first analytics</strong> (generation stays grounded in schema and validated SQL; row-level data is sensitive by default) and a <strong>developer-first embed</strong> (install from npm, bring your own model keys, database connectivity, embeddings, and vector store). The full principles and audience definitions live in{" "}
  <a href="https://github.com/Ygilany/AskDB/blob/main/docs/mission.md">docs/mission.md</a> in the repository.
</p>

## Who you are

<div className="feature-grid">
  <article>
    <h3>Integrator</h3>
    <p>You wire AskDB into a service or internal app: config bootstrap, <code>{"ask()"}</code>, HTTP, or CLI automation.</p>
  </article>
  <article>
    <h3>Schema author</h3>
    <p>You turn introspected DDL into a describable Schema v2 artifact with reviewed descriptions, aliases, and concepts.</p>
  </article>
  <article>
    <h3>Agent or platform owner</h3>
    <p>You need stable headless contracts for tools and CI. MCP is on the roadmap—see <a href="./guides/agents-and-mcp/">Agents and MCP (status)</a>.</p>
  </article>
</div>

## What AskDB gives you

<div className="feature-grid">
  <article>
    <h3>Schema-grounded generation</h3>
    <p>Questions are grounded in an AskDB Schema v2 artifact before SQL is generated and validated.</p>
  </article>
  <article>
    <h3>Bring your own runtime</h3>
    <p>Use any AI SDK-compatible language model. Install <code>{"pg"}</code> only when you choose live Postgres introspection or the pgvector store.</p>
  </article>
  <article>
    <h3>Schema enrichment tools</h3>
    <p><code>{"@askdb/enrich"}</code> powers the TUI and Studio so humans confirm AI-suggested descriptions and aliases.</p>
  </article>
  <article>
    <h3>Retrieval-ready context</h3>
    <p><code>{"@askdb/rag"}</code> chunks the same Schema v2 artifact—no separate knowledge base that can drift from schema truth.</p>
  </article>
</div>

## First run

<p>Install dependencies, create a Schema v2 artifact, then ask a question against it.</p>

```
pnpm install
cp .env.example .env
# or: pnpm exec askdb init  (writes askdb.config.ts; .env guidance is in comments)
pnpm build

pnpm exec askdb introspect \
  --url "$DATABASE_URL" \
  --out my-app.schema \
  --schema-id my-app

pnpm exec askdb enrich --schema my-app.schema

pnpm exec askdb ask \
  --schema my-app.schema \
  --question "Which tables look active?"
```
