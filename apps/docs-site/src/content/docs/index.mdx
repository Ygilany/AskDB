## What AskDB is

<div class="home-north-card">
  <p class="doc-lede home-north-lede">
    AskDB helps developers add natural-language analytics to their own applications. It turns user questions into validated SQL using a schema that your team can inspect, enrich, and control. AskDB does not take over your database—it generates SQL from trusted schema context, validates the query, and leaves execution, approval, permissions, and audit logging in your application. The AI model receives your schema context—table names, column types, and descriptions your team approved—not your actual data. Your rows stay inside your infrastructure. This is what we mean by <strong>trust-first analytics</strong>: answers are grounded, inspectable, and controlled by the application that uses them. For philosophy and limitations, see the{" "}
    <a href="https://github.com/Ygilany/AskDB">AskDB project on GitHub</a>.
  </p>
</div>

## How AskDB works

<p>AskDB uses the same core workflow whether you use the CLI, the Node library, or the HTTP API.</p>

<ol class="home-workflow-steps">
  <li><strong>Initialize AskDB</strong> — Set up AskDB for your project and connect your database, AI model, and optional vector store.</li>
  <li><strong>Create a schema artifact</strong> — Introspect your database so AskDB has a structured view of your tables, columns, relationships, and constraints.</li>
  <li><strong>Ask questions</strong> — Test natural-language questions and inspect the SQL AskDB generates.</li>
  <li><strong>Enrich the schema</strong> — Add descriptions, aliases, business concepts, and common query patterns using Studio or the TUI.</li>
  <li><strong>Add retrieval when needed</strong> — For large schemas, index the schema artifact so AskDB retrieves only the most relevant context per question.</li>
  <li><strong>Embed AskDB where you need it</strong> — Use AskDB through the CLI, Node library, or HTTP API depending on how you want to integrate it.</li>
</ol>

<p>Start with SQL generation from an introspected schema. Add enrichment when you need more business context. Add retrieval when your schema is too large for a single prompt. Expose through HTTP when you want to put it behind your own authentication, permissions, logging, and approval flow.</p>

## What AskDB gives you

<div class="home-value-grid">
  <article>
    <h3>Schema-grounded SQL generation</h3>
    <p>Questions are grounded in your AskDB schema before SQL is generated and validated.</p>
  </article>
  <article>
    <h3>Clear execution boundaries</h3>
    <p>AskDB returns validated SQL. Your application decides whether to show it, review it, approve it, run it, log it, or reject it. AskDB does not execute SQL against your database.</p>
  </article>
  <article>
    <h3>Human-reviewed schema enrichment</h3>
    <p><code>@askdb/enrich</code> powers the TUI and Studio so humans confirm AI-suggested descriptions and aliases.</p>
  </article>
  <article>
    <h3>Retrieval-ready schema context</h3>
    <p><code>@askdb/rag</code> chunks the same schema artifact—no separate knowledge base that can drift from what you ship.</p>
  </article>
  <article>
    <h3>Bring-your-own runtime</h3>
    <p>Use your own model provider, database connection, embeddings, and vector store. Install only the packages your integration needs.</p>
  </article>
</div>

## First run

<p class="home-terminal-intro">Install the CLI, create a schema folder, then ask a question against it.</p>

```bash
npm install askdb
npx askdb init
# add API keys to .env (see comments from init)

npx askdb introspect \
  --url "$DATABASE_URL" \
  --out my-app.schema \
  --schema-id my-app

npx askdb enrich --schema my-app.schema

npx askdb ask \
  --schema my-app.schema \
  --question "Which tables look active?"
```
