<p className="doc-eyebrow">Prisma integration</p>

<p className="doc-lede">The Prisma connector reads <code>{".prisma"}</code> schema files and produces a Schema v2 physical artifact without connecting to a database.</p>

## Install

```
pnpm add @askdb/prisma @askdb/introspect
# add a model provider and dialect when your app calls ask()
pnpm add @askdb/core ai @ai-sdk/openai
```

<p><code>{"@askdb/prisma"}</code> uses <code>{"@prisma/internals"}</code> to parse schema files. It does not connect to a database, does not execute generated SQL, and does not include a SQL dialect. Pair it with <code>{"@askdb/postgres"}</code> for SQL generation against a Postgres target.</p>

## CLI Workflow

```
askdb introspect --engine prisma \
  --prisma-schema ./prisma \
  --out my-app.schema \
  --schema-id my-app

# or print the introspection result without writing files
askdb introspect --engine prisma \
  --prisma-schema ./prisma/schema.prisma \
  --print

# diff against an existing schema directory
askdb introspect --engine prisma \
  --prisma-schema ./prisma \
  --diff my-app.schema
```

<p>Set <code>{"ASKDB_PRISMA_SCHEMA"}</code> (via <code>{"askdb.config.ts"}</code> or <code>{".env"}</code>) to provide a default <code>{"--prisma-schema"}</code> path so you can omit the flag on repeat runs.</p>

## Library API

```
import { introspect } from "@askdb/introspect";
import { createPrismaConnector } from "@askdb/prisma";

await introspect(
  { schemaPath: "./prisma", schemaId: "my-app" },
  { outDir: "my-app.schema", schemaId: "my-app" },
  { connector: createPrismaConnector() },
);
```

<p>The input shape is <code>{"PrismaIntrospectionInput"}</code> — a <code>{"schemaPath"}</code> pointing at a <code>{".prisma"}</code> file or a directory of <code>{".prisma"}</code> files, an optional <code>{"schemaId"}</code>, and optional <code>{"IntrospectionFilters"}</code>.</p>

## Supported Providers

<div className="definition-list">
  <div><code>{"postgresql"}</code><span>Postgres relational tables, columns, relationships, and enums.</span></div>
  <div><code>{"mysql"}</code><span>MySQL relational tables and columns.</span></div>
  <div><code>{"sqlite"}</code><span>SQLite tables and columns.</span></div>
  <div><code>{"sqlserver"}</code><span>SQL Server tables and columns.</span></div>
  <div><code>{"cockroachdb"}</code><span>CockroachDB tables and columns.</span></div>
</div>

<p>MongoDB is not supported — AskDB Schema v2 and SQL generation are relational.</p>

## What It Does Not Have

<p><code>{"@askdb/prisma"}</code> is a file-only connector: no live database connection, no catalog SQL templates, and no SQL dialect. It legitimately omits <code>{"templates()"}</code> from the connector interface. For SQL generation, supply a dialect from an integration package such as <code>{"@askdb/postgres"}</code>.</p>
