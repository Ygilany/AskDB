<p className="doc-eyebrow">Non-Postgres integrations</p>

<p className="doc-lede">Three engine packages follow the same pattern: re-export a dialect constant from <code>{"@askdb/core"}</code>, ship a live introspection connector, and provide an optional driver-backed catalog runner. None currently supports <code>{"--from-export"}</code> or catalog template bundles — unlike <a href="../postgres/">@askdb/postgres</a>.</p>

## MySQL

<p><code>{"@askdb/mysql"}</code> — MySQL and MariaDB integration. Introspects via <code>{"information_schema"}</code>. Exports both <code>{"MYSQL_DIALECT"}</code> (MySQL 8+) and <code>{"MARIADB_DIALECT"}</code> for MariaDB targets.</p>

### Install

```
pnpm add @askdb/core @askdb/introspect @askdb/mysql
# only when using live introspection mode
pnpm add mysql2
```

<p><code>{"mysql2"}</code> is an optional peer dependency.</p>

### What It Provides

<div className="definition-list">
  <div><code>{"MYSQL_DIALECT"}</code><span>Prompting, SQL generation, and validation rules for read-only MySQL <code>{"SELECT"}</code> statements.</span></div>
  <div><code>{"MARIADB_DIALECT"}</code><span>Prompting rules targeting MariaDB — use this instead of <code>{"MYSQL_DIALECT"}</code> when your runtime engine is MariaDB.</span></div>
  <div><code>{"createMysqlCatalogQueryRunner()"}</code><span>An introspection-only catalog query runner backed by <code>{"mysql2"}</code>, lazy-loaded when called.</span></div>
  <div><code>{"createMysqlConnector()"}</code><span>The connector used by <code>{"@askdb/introspect"}</code> for live catalog reads.</span></div>
  <div><code>{"MYSQL_CATALOG_SQL"}</code><span>The raw catalog SQL suite for manual inspection.</span></div>
</div>

### Ask with MySQL

```ts
import { ask, loadSchema } from "@askdb/core";
import { MYSQL_DIALECT } from "@askdb/mysql";

const schema = loadSchema("./my-app.schema");

const { sql } = await ask({
  question: "How many paid orders were created last month?",
  schema,
  model,
  dialect: MYSQL_DIALECT,
});
```

<p>Replace <code>{"MYSQL_DIALECT"}</code> with <code>{"MARIADB_DIALECT"}</code> when targeting MariaDB.</p>

### Introspect with MySQL

```ts
import { introspect } from "@askdb/introspect";
import { createMysqlConnector, createMysqlCatalogQueryRunner } from "@askdb/mysql";

await introspect(
  { mode: "live", runner: createMysqlCatalogQueryRunner(process.env.DATABASE_URL!) },
  { outDir: "./my-app.schema", schemaId: "my-app" },
  { connector: createMysqlConnector() },
);
```

### CLI

```
askdb introspect --engine mysql --url mysql://user:pass@host/db \
  --out my-app.schema --schema-id my-app
```

<p>Set <code>{"ASKDB_INTROSPECT_MYSQL_URL"}</code> (or <code>{"DATABASE_URL"}</code>) to omit <code>{"--url"}</code> on repeat runs.</p>

## SQLite

<p><code>{"@askdb/sqlite"}</code> — SQLite integration. Introspects via <code>{"sqlite_master"}</code> and <code>{"PRAGMA"}</code> functions. Requires SQLite ≥ 3.16 for <code>{"pragma_*"}</code> table-valued functions.</p>

### Install

```
pnpm add @askdb/core @askdb/introspect @askdb/sqlite
# only when using live introspection mode
pnpm add better-sqlite3
```

<p><code>{"better-sqlite3"}</code> is an optional peer dependency.</p>

### What It Provides

<div className="definition-list">
  <div><code>{"SQLITE_DIALECT"}</code><span>Prompting, SQL generation, and validation rules for read-only SQLite <code>{"SELECT"}</code> statements.</span></div>
  <div><code>{"createSqliteCatalogQueryRunner()"}</code><span>An introspection-only catalog query runner backed by <code>{"better-sqlite3"}</code>, lazy-loaded when called.</span></div>
  <div><code>{"createSqliteConnector()"}</code><span>The connector used by <code>{"@askdb/introspect"}</code> for live catalog reads.</span></div>
  <div><code>{"SQLITE_CATALOG_SQL"}</code><span>The raw catalog SQL suite for manual inspection.</span></div>
</div>

### Ask with SQLite

```ts
import { ask, loadSchema } from "@askdb/core";
import { SQLITE_DIALECT } from "@askdb/sqlite";

const schema = loadSchema("./my-app.schema");

const { sql } = await ask({
  question: "How many paid orders were created last month?",
  schema,
  model,
  dialect: SQLITE_DIALECT,
});
```

### Introspect with SQLite

```ts
import { introspect } from "@askdb/introspect";
import { createSqliteConnector, createSqliteCatalogQueryRunner } from "@askdb/sqlite";

await introspect(
  { mode: "live", runner: createSqliteCatalogQueryRunner("/path/to/database.db") },
  { outDir: "./my-app.schema", schemaId: "my-app" },
  { connector: createSqliteConnector() },
);
```

### CLI

```
askdb introspect --engine sqlite --url /path/to/database.db \
  --out my-app.schema --schema-id my-app
```

<p>Set <code>{"ASKDB_INTROSPECT_SQLITE_FILE"}</code> to omit <code>{"--url"}</code> on repeat runs. Unlike other engines, <code>{"DATABASE_URL"}</code> is not used as a fallback for SQLite.</p>

## SQL Server

<p><code>{"@askdb/sqlserver"}</code> — Microsoft SQL Server integration. Introspects via <code>{"sys.*"}</code> catalog views.</p>

### Install

```
pnpm add @askdb/core @askdb/introspect @askdb/sqlserver
# only when using live introspection mode
pnpm add mssql
```

<p><code>{"mssql"}</code> is an optional peer dependency.</p>

### What It Provides

<div className="definition-list">
  <div><code>{"SQLSERVER_DIALECT"}</code><span>Prompting, SQL generation, and validation rules for read-only SQL Server <code>{"SELECT"}</code> statements (T-SQL).</span></div>
  <div><code>{"createSqlServerCatalogQueryRunner()"}</code><span>An introspection-only catalog query runner backed by <code>{"mssql"}</code>, lazy-loaded when called.</span></div>
  <div><code>{"createSqlServerConnector()"}</code><span>The connector used by <code>{"@askdb/introspect"}</code> for live catalog reads.</span></div>
  <div><code>{"SQLSERVER_CATALOG_SQL"}</code><span>The raw catalog SQL suite for manual inspection.</span></div>
</div>

### Ask with SQL Server

```ts
import { ask, loadSchema } from "@askdb/core";
import { SQLSERVER_DIALECT } from "@askdb/sqlserver";

const schema = loadSchema("./my-app.schema");

const { sql } = await ask({
  question: "How many paid orders were created last month?",
  schema,
  model,
  dialect: SQLSERVER_DIALECT,
});
```

### Introspect with SQL Server

```ts
import { introspect } from "@askdb/introspect";
import { createSqlServerConnector, createSqlServerCatalogQueryRunner } from "@askdb/sqlserver";

await introspect(
  {
    mode: "live",
    runner: createSqlServerCatalogQueryRunner(
      "Server=host,1433;Database=mydb;User Id=sa;Password=pass;",
    ),
  },
  { outDir: "./my-app.schema", schemaId: "my-app" },
  { connector: createSqlServerConnector() },
);
```

### CLI

```
askdb introspect --engine sqlserver \
  --url "Server=host,1433;Database=mydb;User Id=sa;Password=pass;" \
  --out my-app.schema --schema-id my-app
```

<p>Set <code>{"ASKDB_INTROSPECT_SQLSERVER_URL"}</code> (or <code>{"DATABASE_URL"}</code>) to omit <code>{"--url"}</code> on repeat runs.</p>
