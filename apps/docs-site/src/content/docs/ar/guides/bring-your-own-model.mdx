## طريقتان لربط نموذج

| | ماذا تفعل | الأفضل لـ |
| --- | --- | --- |
| **في إعدادك** | عيّن `ai.provider` في `askdb.config.ts`. تحلّ CLI وStudio وواجهة HTTP البرمجية النموذج نيابةً عنك. لا كود للنموذج. | سير عمل CLI، وStudio، وخدمة HTTP المجمّعة |
| **في كودك** | مرّر أي `LanguageModel` من Vercel AI SDK إلى `ask()`. تحكّم كامل. | تضمين `@askdb/core` في خدمتك الخاصة |

نفس المزوّدين، ونفس المفاتيح — اختر لكل سطح. يمكنك استخدام كليهما في مشروع واحد (على سبيل المثال: تستخدم Studio إعدادك، بينما تبني خدمة التضمين لديك النموذج مباشرةً).

## في إعدادك (CLI، Studio، واجهة HTTP البرمجية)

عيّن `ai.provider` في `askdb.config.ts` وقدّم متغيّر البيئة المطابق. تستدعي CLI وStudio وخدمة HTTP الدالة `createLanguageModelFromEnv` داخليًا.

## في كودك (التضمين)

عندما تضمّن `@askdb/core` مباشرةً، مرّر أي `LanguageModel` من Vercel AI SDK إلى `ask()`. يستدعي AskDB النموذج بالمُوجِّه المُجمَّع ويحلّل SQL الذي يعيده. لا وكيل. لا وسيط.

```ts
import { ask } from "@askdb/core";

const { sql } = await ask({
  question,
  schema,
  dialect: "postgres",
  model, // any LanguageModel from the Vercel AI SDK
});
```

ثبّت حزمة المزوّد وابنِ النموذج:

## إعداد واحد يقود كليهما

إذا كنت تضمّن `@askdb/core` لكنك تريد إعادة استخدام نفس `askdb.config.ts` بدلًا من ترميز مزوّد مباشرةً، فاستخدم سجلّ الذكاء الاصطناعي. هذه هي بالضبط كيفية حلّ CLI وواجهة HTTP البرمجية المجمّعة للنماذج داخليًا.

```bash
npm install @askdb/config @askdb/ai @askdb/ai-openai @askdb/ai-anthropic
```

```ts
import { bootstrapAskDbEnv, getAskDbRuntimeConfig } from "@askdb/config";
import { createAiRegistry } from "@askdb/ai";
import { openaiProvider } from "@askdb/ai-openai";
import { anthropicProvider } from "@askdb/ai-anthropic";

bootstrapAskDbEnv({ cwd: process.cwd() });
const rt = getAskDbRuntimeConfig();

const ai = createAiRegistry([openaiProvider, anthropicProvider]);
const model = await ai.createLanguageModelFromEnv(rt.ai.aiEnv);
```

تعلن المُحوِّلات `ai` و`@askdb/ai` كاعتماديات قرينة — أضف `ai` إلى `package.json` الخاص بك عندما تريد تثبيت إصدارها.

## إدارة المفاتيح

يقيم مفتاح API الخاص بنموذجك في بيئتك، لا في أداة المخطّط ولا في AskDB. تحمّل CLI ملف `.env` للعمل المحلي؛ وفي الإنتاج، قدّم المفتاح بنفس الطريقة التي تقدّم بها عنوان URL لقاعدة البيانات.

لا يملك AskDB واجهة برمجية خاصة به للمصادقة — إنه مكتبة. الاعتمادات الوحيدة التي يمسّها هي تلك التي تمرّرها إلى مزوّد نموذجك عبر AI SDK.

## تبديل النماذج

يمكن تبديل النماذج دون تغيير أي كود آخر في AskDB. نمط شائع: نموذج أرخص للتحليلات منخفضة المخاطر، ونموذج أقوى للاستعلامات المعقّدة متعدّدة الربط.

```ts
import { openai } from "@ai-sdk/openai";

const cheapModel = openai("gpt-4o-mini");
const strongModel = openai("gpt-4o");

const model = question.length > 200 ? strongModel : cheapModel;

const { sql } = await ask({ question, schema, dialect: "postgres", model });
```

## التكلفة

يفوترك مزوّد نموذجك عن كل استدعاء `ask()`. تتناسب التكلفة مع:

- حجم المخطّط (الأدوات الأكبر تعني المزيد من الرموز في المُوجِّه) — استخدم [RAG للمخطّطات الكبيرة](/ar/guides/rag-for-large-schemas/) لإرسال الشريحة ذات الصلة فقط. يتولّى AskDB التجزئة والتضمين والاسترجاع؛ تضبط المخزن وهو يتكفّل بالباقي.
- طول السؤال.
- فئة النموذج.

AskDB نفسه مرخّص بموجب Apache 2.0 ومجاني.
