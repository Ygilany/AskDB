<p class="doc-eyebrow">العقد</p>

<p class="doc-lede">مُصنَّع المخطّط هو الملف الواحد (أو الدليل) الذي يكتبه فريقك ويراجعه ويُدرجه في التحكّم بالإصدار. إنه السياق الذي يستخدمه النموذج لتوليد SQL — وهو ملكك لتطوّره كأي قطعة أخرى من الكود المصدري.</p>

## على القرص

مُصنَّع المخطّط هو مجلّد يحتوي على ملف واحد مطلوب وعدد قليل من الملفات المرافقة الاختيارية:

```
my-app.schema/
  schema.json           # required — physical structure
  tables/
    users.md            # optional — per-table descriptions, aliases, concepts
    orders.md
  concepts.md           # optional — cross-table business concepts
  schema.lock.json      # optional — bundle integrity
```

`schema.json` فقط هو المطلوب. المُصنَّع الذي لا يحتوي على أي markdown صالح؛ يمكنك إضافة الإثراء تدريجيًا.

عندما تحتاج إلى ملف واحد قابل للنقل (لتوزيعه على خدمة، أو وكيل، أو مُصنَّع بناء)، احزم الدليل:

```bash
npx askdb bundle my-app.schema --out my-app.schema.bundle.json
```

## الطبقة المادية

يحتوي `schema.json` على الحقائق البنيوية: المعرّفات المستقرة، وأسماء الجداول والأعمدة المادية، والأنواع، وقابلية القيمة الفارغة، والمفاتيح الأساسية، والعلاقات، وأعلام الحساسية.

```json
{
  "version": 2,
  "schemaId": "my-app",
  "tables": [
    {
      "id": "table:users",
      "name": "users",
      "columns": [
        {
          "id": "table:users#email",
          "name": "email",
          "type": "text",
          "nullable": false,
          "sensitive": true
        }
      ]
    }
  ]
}
```

تُعاد الطبقة المادية بواسطة `askdb introspect` — أنت لا تحرّرها يدويًا.

## طبقة الإثراء

يضيف markdown لكل جدول المعنى الذي يحتاجه النموذج لتوليد SQL جيد: الأوصاف، والأسماء البديلة، ولغة الاستعلام الشائعة، وأسئلة الأمثلة، والملاحظات على مستوى الأعمدة.

```markdown
---
id: table:orders
name: orders
schemaId: my-app
aliases: [purchases, sales]
columns:
  - id: table:orders#status
    enum: [pending, paid, shipped, cancelled]
    description: Order lifecycle state.
---

# Table: orders

Customer purchase orders. One row per submitted order.

## Common query language

- "sales" usually means paid orders
- "active customers" excludes status = cancelled
```

تبقى هذه الطبقة بعد إعادة الاستبطان. عندما تتغيّر قاعدة بياناتك، يُحدّث `askdb introspect` ملف `schema.json` ويترك markdown الخاص بك سليمًا.

## ما يتلقّاه النموذج فعليًا

النموذج لا يرى أبدًا الدليل الذي تكتبه. في وقت التوليد، يدمج AskDB الطبقة المادية والإثراء في نص واحد مدمج موجّه إلى النموذج — وصف بأسلوب DDL حيث يُعرَض كل جدول ككتلة ويُطوى إثراؤك كتعليقات مُضمَّنة (الأوصاف، والأسماء البديلة، و"لغة الاستعلام الشائعة" المرفقة بالجدول والأعمدة التي تصفها). تُوسَم الحقول الحساسة أو تُحجَب هنا وفقًا للوضع الذي تختاره.

إذن الطبقتان اللتان تحافظ عليهما بشكل منفصل على القرص — `schema.json` للبنية، و markdown للمعنى — تُجمَّعان في وصف واحد مُعلَّق للمخطّط يقرأه النموذج. ذلك النص المُعاد تغليفه، إضافةً إلى السؤال، هو المُحفِّز بأكمله: ليس صفوفك أبدًا، ولا بيانات اعتمادك، ولا نتائج استعلامك. عندما يكون المخطّط أكبر من أن يُرسَل بالكامل، يُطبَّق العرض نفسه على الشريحة ذات الصلة فقط المسترجَعة للسؤال (راجع [RAG للمخطّطات الكبيرة](/ar/guides/rag-for-large-schemas/)).

## المعرّفات المستقرة

كل جدول وعمود ومفهوم له معرّف مستقر. إعادة تسمية عمود في قاعدة البيانات لا تكسر الإثراء الذي يشير إليه — يبقى المعرّف كما هو.

| المعرّف | يشير إلى |
| --- | --- |
| `table:users` | جدول |
| `table:users#email` | عمود |
| `concept:customer` | مفهوم عمل عابر للجداول |

## لماذا يهمّ هذا

مُصنَّع المخطّط هو خيار مقصود. البديل — قالب مُحفِّز مدفون في كود التطبيق — غامض، يصعب مراجعته، ومتشابك مع بقية شيفرتك المصدرية.

الملف القابل للإصدار يعني:

- يمكن لفريق البيانات لديك مراجعة التغييرات في طلب سحب (PR) كأي كود آخر.
- يمكنك أن ترى، سطرًا بسطر، السياق الذي يتلقّاه النموذج.
- يتطوّر الإثراء بوتيرته الخاصة، منفصلاً عن عمليات نشر التطبيق.
- المُصنَّع نفسه يشغّل CLI، والمكتبة، وواجهة HTTP API — مصدر واحد للحقيقة.
