<p class="doc-lede">AskDB هو دالة تحوّل سؤالاً ومُصنَّع المخطّط إلى SQL مُتحقَّق منه. كل شيء آخر — التنفيذ، الموافقة، التدقيق — يحدث داخل تطبيقك، وليس داخل AskDB.</p>

## الخطوات الخمس

عندما تستدعي `ask()` (أو واجهة CLI / HTTP المكافئة)، ينفّذ AskDB هذه الخطوات بالترتيب:

1. **تحميل مُصنَّع المخطّط.** دليل، أو ملف حزمة، أو `schema.json` مُحلَّل — أيًّا كان ما تقدّمه.
2. **استرجاع المقاطع ذات الصلة (اختياري).** بالنسبة للمخطّطات الكبيرة، يسحب AskDB فقط الشريحة من المُصنَّع ذات الصلة بالسؤال، باستخدام `@askdb/rag`. المخطّطات الأصغر تتخطّى هذه الخطوة.
3. **بناء المُحفِّز (prompt).** يتم تجميع البيانات الوصفية المادية، وأي أوصاف للجداول كتبتها، ولهجة SQL المختارة في السياق الذي يراه النموذج.
4. **التوليد.** يستدعي AskDB النموذج الذي وفّرته — أي مزوّد متوافق مع [Vercel AI SDK](/ar/guides/bring-your-own-model/) — ويُعيد النموذج SQL. لا يستضيف AskDB الاستدلال؛ مفتاحك، وفوترتك.
5. **التحقّق.** يُفحَص SQL المُعاد مقابل حواجز الحماية التي يفرضها AskDB: عبارة واحدة، يجب أن تبدأ بـ `SELECT` أو `WITH`، بلا كلمات مفتاحية DDL/DML، و — عندما يعلن مُصنَّعك سياسة مستأجر — المُسنَدات المطلوبة للمستأجر. القائمة الكاملة والفحوصات الدقيقة موجودة في [حدود الأمان](/ar/concepts/safety-boundaries/). يُرفَض SQL غير الصالح، ولا يُعاد مع تحذير.

القيمة المُعادة هي سلسلة من SQL المُتحقَّق منه وقائمة معاملات. ما يحدث بعد ذلك قرارك أنت.

## حدّ وقت التشغيل

<img src={runtimeBoundaryUrl} alt="Flow from question and schema through ask() to validated SQL, with execution owned by your application" class="doc-diagram" />

يتوقف AskDB عند SQL المُتحقَّق منه. تطبيقك — الكود الذي يستورد `@askdb/core` أو يستدعي واجهة HTTP API — يملك كل شيء بعد ذلك:

- مشغّل قاعدة البيانات وتجمّع الاتصالات.
- الأدوار للقراءة فقط وأي أمان إضافي على مستوى الصفوف.
- مهلات الاستعلام، وحدود الصفوف، وترقيم صفحات النتائج.
- ما إذا كان سيُعرض SQL للمستخدم، أو يُسجَّل، أو يُوضَع في قائمة انتظار للمراجعة، أو يُشغَّل فورًا.
- تسجيل التدقيق وقابلية الملاحظة.

هذا الفصل مقصود. AskDB هو خطوة لتوليد الكود، وليس وقت تشغيل. هذا ما يجعله آمنًا للربط بالخدمات الموجودة دون إعادة بناء نموذج الأمان الخاص بك.

## أين يوجد النموذج

أنت تأتي بالنموذج. يستدعي AskDB أي مزوّد تربطه — عادةً عبر Vercel AI SDK — باستخدام مفتاح API الخاص بك. تذهب الطلبات مباشرةً من بيئتك إلى المزوّد. لا يقوم AskDB بالوكالة عنها، أو تسجيلها، أو تخزينها مؤقتًا.

النموذج نفسه لا يرى أبدًا اتصال قاعدة بياناتك، أو بيانات اعتمادك، أو الصفوف التي يُعيدها تطبيقك. راجع [نموذج الخصوصية](/ar/concepts/privacy-model/) للحصول على القائمة الدقيقة لما يعبر الحدّ.
