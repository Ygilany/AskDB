<p className="doc-eyebrow">Packages and boundaries</p>

<p className="doc-lede">AskDB is split into focused packages so integrators can choose the exact SQL generation, introspection, enrichment, and retrieval surfaces they need.</p>

## Runtime Boundary

<p><code>{"@askdb/core"}</code> loads your AskDB schema, builds model prompts, calls the supplied model, and returns validated SQL. It does not own database drivers or generated-SQL execution. The host application owns approval, execution, database roles, tenant policy, network controls, and audit logging.</p>

## Env bootstrap

<p>First-party Node entrypoints call <code>{"bootstrapAskDbEnv()"}</code> from <code>{"@askdb/config"}</code> (after loading <code>{".env"}</code> where the binary wires dotenv candidate paths). That evaluates <code>{"askdb.config.*"}</code> / <code>{".config/askdb.*"}</code> for the chosen working directory and installs the <strong>runtime snapshot</strong>. Downstream code reads <code>{"getAskDbRuntimeConfig()"}</code>; pass <code>{"getAskDbRuntimeConfig().ai.aiEnv"}</code> into <code>{"@askdb/core"}</code> helpers that take an explicit env map. See <a href="../environment/">Environment</a> for the operator view; rationale lives in <a href="https://github.com/Ygilany/AskDB/blob/main/docs/adrs/0005-askdb-config-and-env-bootstrap.md">ADR 0005 (GitHub)</a>.</p>

## Dependency Direction

<p>Reusable packages stay small and layered. First-party surfaces compose them into workflows. Arrows point <strong>toward the dependency</strong> (children depend on parents).</p>

## Connectors vs Peers

<div className="definition-list">
  <div><code>{"Connector package"}</code><span>An AskDB integration that describes a schema source, such as <code>{"@askdb/postgres"}</code> or <code>{"@askdb/prisma"}</code>.</span></div>
  <div><code>{"Optional peer"}</code><span>A runtime dependency installed only when a selected feature needs it, such as <code>{"pg"}</code> for live Postgres introspection or pgvector.</span></div>
  <div><code>{"Provider package"}</code><span>A model or embedding integration selected by the host, such as <code>{"ai"}</code> and <code>{"@ai-sdk/openai"}</code>.</span></div>
</div>

## Install Profiles

<div className="definition-list">
  <div><code>{"SQL generation"}</code><span>Install <code>{"@askdb/core"}</code>, a dialect package such as <code>{"@askdb/postgres"}</code>, and your model provider. <code>{"pg"}</code> is not required.</span></div>
  <div><code>{"Live Postgres introspection"}</code><span>Install <code>{"@askdb/introspect"}</code>, <code>{"@askdb/postgres"}</code>, and <code>{"pg"}</code> if using the bundled catalog runner.</span></div>
  <div><code>{"Air-gapped Postgres introspection"}</code><span>Install <code>{"@askdb/introspect"}</code> and <code>{"@askdb/postgres"}</code>. Use from-export bundles without <code>{"pg"}</code>.</span></div>
  <div><code>{"Prisma introspection"}</code><span>Install <code>{"@askdb/introspect"}</code> and <code>{"@askdb/prisma"}</code>. It reads schema files and does not include a SQL dialect.</span></div>
  <div><code>{"Enrichment"}</code><span>Use <code>{"@askdb/tui"}</code> or <code>{"@askdb/studio"}</code> for maintained UIs, or <code>{"@askdb/enrich"}</code> for custom authoring surfaces.</span></div>
  <div><code>{"RAG"}</code><span>Install <code>{"@askdb/rag"}</code>; add embedding providers and <code>{"pg"}</code> only for the adapters you choose.</span></div>
</div>

## Connector Matrix

<div className="definition-list">
  <div><code>{"@askdb/postgres"}</code><span>Live catalog reads, from-export bundles, catalog SQL templates, optional <code>{"pg"}</code> runner, and <code>{"postgresDialect"}</code>.</span></div>
  <div><code>{"@askdb/prisma"}</code><span>Offline <code>{"schema.prisma"}</code> or Prisma directory reads. No live database, no templates, and no dialect.</span></div>
</div>

## Extension Points

- <code>{"AskDialect"}</code> for SQL generation and validation policy.
- <code>{"Connector<TInput>"}</code> for schema metadata sources.
- <code>{"CatalogQueryRunner"}</code> for alternate live database clients.
- <code>{"@askdb/enrich"}</code> workspace helpers for custom authoring UIs.
- <code>{"Embedder"}</code>, <code>{"VectorStore"}</code>, and <code>{"Retriever"}</code> for retrieval over your AskDB schema.

The diagrams on this page are the canonical illustrations for integrators. If you contribute to AskDB itself, you may also find narrative and Mermaid versions in the <a href="https://github.com/Ygilany/AskDB/blob/main/docs/architecture.md">architecture write-up (GitHub)</a>.
