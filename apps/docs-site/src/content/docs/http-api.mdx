<p className="doc-lede">The HTTP API exposes a minimal server around @askdb/core. POST /ask returns validated SQL only.</p>

## When to use

<ul>
  <li>A browser or remote service should call AskDB over <strong>HTTP</strong> instead of linking <code>{"@askdb/core"}</code> directly.</li>
  <li>You will mount your own <strong>auth, TLS, and rate limits</strong> in front—see <a href="../guides/http-api/">Run the HTTP API</a>.</li>
</ul>

## Run Locally



```
pnpm -C apps/http-api build
node apps/http-api/dist/bin.js
```


          <p>Set <code>{"ASKDB_SCHEMA_PATH"}</code> in the repo-root <code>{".env"}</code> and/or in nested fields under <code>{"askdb.config.*"}</code> for the process working directory. The binary runs <code>{"bootstrapAskDbEnv()"}</code>: it loads the first existing <code>{".env"}</code> among repo root, current working directory, and the http-api package directory, then evaluates <code>{"askdb.config.*"}</code> / <code>{".config/askdb.*"}</code> from <code>{"cwd"}</code> and installs the AskDB runtime snapshot. The server reads settings with <code>{"getAskDbRuntimeConfig()"}</code>; AskDB does not merge the full config into <code>{"process.env"}</code>.</p>

## Health Check



```
curl -sS http://127.0.0.1:3000/health
```



## Ask Endpoint



```
curl -sS http://127.0.0.1:3000/ask \
  -H 'content-type: application/json' \
  -H 'x-correlation-id: demo-123' \
  -d '{
    "question": "How many users are there?"
  }'
```



## Execution Boundary

<p>Execution is not part of this API. Requests using the retired execution controls receive <code>{"400"}</code>. Review generated SQL and run any approved query outside AskDB under your own database roles, read-only controls, tenant policy, and audit logging.</p>
