<p class="doc-eyebrow">Install</p>

<p class="doc-lede">AskDB ships as a set of focused npm packages. Install the CLI for local work and authoring; add the core library and an engine adapter when you're ready to embed AskDB in your own service.</p>

## Requirements

- Node 20 or newer.
- An API key from a supported model provider — OpenAI, Anthropic, Google, OpenRouter, or any OpenAI-compatible endpoint.
- The database you want to ask questions of — or a Prisma schema file.

## The CLI

The CLI covers introspection, enrichment, schema validation, and local serving. Install it globally or per project.

<Tabs syncKey="pkg">
  <TabItem label="npm">

```bash
npm install askdb
npx askdb --help
```

  </TabItem>
  <TabItem label="pnpm">

```bash
pnpm add askdb
pnpm dlx askdb --help
```

  </TabItem>
  <TabItem label="yarn">

```bash
yarn add askdb
yarn dlx askdb --help
```

  </TabItem>
</Tabs>

Run `npx askdb init` once at the root of your project to scaffold `askdb.config.ts`. The template includes comments listing the environment variables each provider needs — copy the relevant ones into a `.env` you create manually.

## Embedding in a Node app

To call AskDB from your own TypeScript or JavaScript code, install the core library and the engine adapter for your database.

<Tabs syncKey="pkg">
  <TabItem label="npm">

```bash
# Core library — dialect-agnostic SQL generation
npm install @askdb/core

# Pick the engine adapter for your database
npm install @askdb/postgres      # PostgreSQL
# npm install @askdb/mysql       # MySQL
# npm install @askdb/sqlite      # SQLite
# npm install @askdb/sqlserver   # SQL Server
```

  </TabItem>
  <TabItem label="pnpm">

```bash
# Core library — dialect-agnostic SQL generation
pnpm add @askdb/core

# Pick the engine adapter for your database
pnpm add @askdb/postgres         # PostgreSQL
# pnpm add @askdb/mysql          # MySQL
# pnpm add @askdb/sqlite         # SQLite
# pnpm add @askdb/sqlserver      # SQL Server
```

  </TabItem>
  <TabItem label="yarn">

```bash
# Core library — dialect-agnostic SQL generation
yarn add @askdb/core

# Pick the engine adapter for your database
yarn add @askdb/postgres         # PostgreSQL
# yarn add @askdb/mysql          # MySQL
# yarn add @askdb/sqlite         # SQLite
# yarn add @askdb/sqlserver      # SQL Server
```

  </TabItem>
</Tabs>

A typical project installs `@askdb/core` plus one engine. See [Switch engines](/guides/switch-engines/) for what differs per engine.

## Model provider

AskDB doesn't bundle inference. Bring any provider that speaks the expected interface — your key, your billing.

<Tabs syncKey="pkg">
  <TabItem label="npm">

```bash
# Example: Vercel AI SDK with OpenAI
npm install @ai-sdk/openai

# Optional: AskDB config/env model factory and a provider adapter (pick one)
npm install @askdb/ai @askdb/ai-openai      # OpenAI
# npm install @askdb/ai @askdb/ai-anthropic  # Anthropic
# npm install @askdb/ai @askdb/ai-azure      # Azure OpenAI
# npm install @askdb/ai @askdb/ai-google     # Google Gemini
```

  </TabItem>
  <TabItem label="pnpm">

```bash
# Example: Vercel AI SDK with OpenAI
pnpm add @ai-sdk/openai

# Optional: AskDB config/env model factory and a provider adapter (pick one)
pnpm add @askdb/ai @askdb/ai-openai         # OpenAI
# pnpm add @askdb/ai @askdb/ai-anthropic    # Anthropic
# pnpm add @askdb/ai @askdb/ai-azure        # Azure OpenAI
# pnpm add @askdb/ai @askdb/ai-google       # Google Gemini
```

  </TabItem>
  <TabItem label="yarn">

```bash
# Example: Vercel AI SDK with OpenAI
```

Adapters declare `ai` and `@askdb/ai` as peer dependencies. Add `ai` to your own `package.json` when you want to pin its version.

The full list of supported providers lives in [Bring your own model](/guides/bring-your-own-model/).

## Authoring the schema

Enrichment turns a raw introspected schema into something the model can generate good SQL against. Install whichever surface you want to use.

<Tabs syncKey="pkg">
  <TabItem label="npm">

```bash
# Browser UI (recommended)
npm install @askdb/studio

# Terminal UI — keyboard-driven, works over SSH
# npm install @askdb/tui

# Programmatic API — for building custom enrichment UIs
# npm install @askdb/enrich
```

  </TabItem>
  <TabItem label="pnpm">

The CLI ships with `studio` and `enrich` commands that use these packages under the hood, so you only need to install them explicitly if you're embedding the authoring surfaces in your own app. See [Author your schema](/guides/author-your-schema/) for the workflow.

## Introspection

The CLI handles introspection for the standard engines out of the box. Install these packages directly only when you want to call introspection from your own code, or use the Prisma schema-file introspector.
