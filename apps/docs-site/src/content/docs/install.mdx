<p class="doc-eyebrow">Install</p>

<p class="doc-lede">
  AskDB ships as a set of focused npm packages. Install the CLI for
  local work and authoring; add the core library and an engine
  adapter when you're ready to embed AskDB in your own service.
</p>

## Requirements

- Node 20 or newer.
- An API key from a supported model provider — OpenAI, Anthropic,
  Google, OpenRouter, or any OpenAI-compatible endpoint.
- The database you want to ask questions of — or a Prisma schema file.

## The CLI

The CLI covers introspection, enrichment, schema validation, and
local serving. Install it globally or per project.

```bash
npm install askdb
npx askdb --help
```

Run `npx askdb init` once at the root of your project to scaffold
`askdb.config.ts`. The template includes comments listing the
environment variables each provider needs — copy the relevant ones
into a `.env` you create manually.

## Embedding in a Node app

To call AskDB from your own TypeScript or JavaScript code, install
the core library and the engine adapter for your database.

```bash
# Core library — dialect-agnostic SQL generation
npm install @askdb/core

# Pick the engine adapter for your database
npm install @askdb/postgres      # PostgreSQL
npm install @askdb/mysql         # MySQL
npm install @askdb/sqlite        # SQLite
npm install @askdb/sqlserver     # SQL Server
```

A typical project installs `@askdb/core` plus one engine. See [Switch
engines](./guides/switch-engines/) for what differs per engine.

## Model provider

AskDB doesn't bundle inference. Bring any provider that speaks the
expected interface — your key, your billing.

```bash
# Example: Vercel AI SDK with OpenAI
npm install ai @ai-sdk/openai
```

The full list of supported providers lives in [Bring your own
model](./guides/bring-your-own-model/).

## Authoring the schema

Enrichment turns a raw introspected schema into something the model
can generate good SQL against. Install whichever surface you want to
use.

```bash
# Browser UI (recommended)
npm install @askdb/studio

# Terminal UI — keyboard-driven, works over SSH
npm install @askdb/tui

# Programmatic API — for building custom enrichment UIs
npm install @askdb/enrich
```

The CLI ships with `studio` and `enrich` commands that use these
packages under the hood, so you only need to install them explicitly
if you're embedding the authoring surfaces in your own app. See
[Author your schema](./guides/author-your-schema/) for the workflow.

## Introspection

The CLI handles introspection for the standard engines out of the
box. Install these packages directly only when you want to call
introspection from your own code, or use the Prisma schema-file
introspector.

```bash
# Programmatic introspection
npm install @askdb/introspect

# Read a Prisma schema file directly — no live database required
npm install @askdb/prisma
```

## Retrieval (RAG)

For schemas too large to fit in a single prompt, AskDB indexes the
schema artifact and retrieves only the relevant slice per question.

```bash
npm install @askdb/rag
```

The full walkthrough — including the recommended pgvector store —
lives in [RAG for large schemas](./guides/rag-for-large-schemas/).

## HTTP service

To expose AskDB over HTTP — for Python clients, browsers, or agent
frameworks — install the HTTP package and follow [Deploy as HTTP
service](./guides/deploy-as-http-service/).

```bash
npm install @askdb/http-api
```
