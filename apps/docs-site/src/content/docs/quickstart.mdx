<p class="doc-eyebrow">From install to first query</p>

<p class="doc-lede">
  Install the AskDB CLI, point it at your database, optionally enrich the
  generated schema artifact, and ask your first question. The same
  packages drop straight into a Node service when you're ready to embed.
</p>

## Prerequisites

- Node 20 or newer.
- npm, pnpm, or yarn (examples use `npx`).
- An API key for any supported model provider — OpenAI (or any
  OpenAI-compatible endpoint) or Azure OpenAI.
- A database AskDB can read — Postgres, MySQL, SQLite, or SQL Server —
  or a Prisma schema file (no live database required).

## 1. Install the CLI

```bash
npm install askdb
npx askdb init
```

`askdb init` scaffolds an `askdb.config.ts` and an `.env` with the
required keys. Fill those in before continuing.

## 2. Introspect your database

Introspection produces an **AskDB schema artifact** — a directory on
disk that captures your tables, columns, types, and relationships.
That artifact is the only thing the model ever sees about your
database.

### From a live database

```bash
npx askdb introspect \
  --url "$DATABASE_URL" \
  --out my-app.schema \
  --schema-id my-app
```

Swap in any supported engine; AskDB picks the right dialect from the
connection string. See [Switch
engines](./guides/switch-engines/) for the install
matrix.

### From a Prisma schema

If you already have a Prisma schema file, introspect directly from it
— no live database required.

```bash
npx askdb introspect \
  --engine prisma \
  --prisma-schema ./prisma/schema.prisma \
  --out my-app.schema \
  --schema-id my-app
```

## 3. Enrich the schema

Raw introspection captures structure. Enrichment adds the layer that
makes generation good: descriptions, aliases, business concepts,
sensitive markers, and example questions. AskDB ships two authoring
surfaces — pick whichever fits your workflow.

**Studio** (recommended) — a browser UI served locally:

```bash
npx askdb studio --schema my-app.schema
```

**TUI** — keyboard-driven, works in any terminal (including SSH):

```bash
npx askdb enrich --schema my-app.schema
```

Both write the same files on disk. The full walkthrough lives in
[Author your schema](./guides/author-your-schema/).

## 4. Ask your first question

```bash
npx askdb ask \
  --schema my-app.schema \
  --question "Which customers signed up last week?"
```

AskDB returns validated SQL. It does **not** run it against your
database — your application owns execution.
