<p className="doc-eyebrow">From clone to first query</p>

<p className="doc-lede">Build the workspace, generate or use a schema artifact, enrich it, and run AskDB from the CLI. The same pieces can then move into your application code.</p>

## Prerequisites

<ul>
            <li>Node 20 or newer.</li>
            <li>pnpm 11.</li>
            <li>An API key for a supported AI provider — OpenAI (or any OpenAI-compatible endpoint) or Azure OpenAI — for live NL-to-SQL generation.</li>
            <li>A PostgreSQL connection string only if you want live Postgres introspection. A Prisma schema file is enough if you use <code>{"--engine prisma"}</code>.</li>
          </ul>

## Use the Bundled Fixture

<p>The fastest path uses the sample Schema v2 directory committed under <code>{"fixtures/schemas/orders-users.schema"}</code>.</p>


```
pnpm install
cp .env.example .env
# edit .env for local secrets. This repo also ships a root askdb.config.ts;
# or run `pnpm exec askdb init` in a new project (comments document .env keys).
pnpm build

pnpm exec askdb ask \
  --schema fixtures/schemas/orders-users.schema \
  --question "How many orders are there?"
```



## Create a Schema from a Database

### From Postgres

<p>For a live Postgres database, generate a Schema v2 directory first, then point AskDB at it.</p>


```
pnpm exec askdb introspect \
  --url "$DATABASE_URL" \
  --out my-app.schema \
  --schema-id my-app

pnpm exec askdb enrich --schema my-app.schema

pnpm exec askdb ask \
  --schema my-app.schema \
  --question "Which tables are connected to users?"
```



### From a Prisma Schema

<p>If you already have a Prisma schema file, introspect directly from it — no live database required.</p>


```
pnpm exec askdb introspect \
  --engine prisma \
  --prisma-schema ./prisma/schema.prisma \
  --out my-app.schema \
  --schema-id my-app

pnpm exec askdb ask \
  --schema my-app.schema \
  --question "Which tables are connected to users?"
```


<p>Set <code>{"ASKDB_PRISMA_SCHEMA"}</code> in <code>{".env"}</code> to make <code>{"--prisma-schema"}</code> optional on repeat runs. See <a href="../prisma/">@askdb/prisma</a> for supported providers.</p>



## Enrich the Schema

<p>Raw introspection gives AskDB physical metadata. The enrichment step adds the describable layer: table descriptions, aliases, column notes, common query language, example questions, and concepts. AskDB ships two enrichment UIs — pick whichever suits your workflow.</p>

**Terminal UI** — keyboard-driven, works in any terminal:

```
pnpm exec askdb enrich --schema my-app.schema
```

**Studio** — browser-based UI served locally at <code>http://127.0.0.1:5556</code>:

```
pnpm exec askdb studio --schema my-app.schema
```

<p>Both UIs use <code>{"@askdb/enrich"}</code> under the hood and produce the same describable layer on disk. Once enriched, bundle the directory into a single portable artifact:</p>

```
# produce a single read-only artifact for distribution
pnpm exec askdb bundle my-app.schema --out my-app.schema.bundle.json
```



## SQL Output Only

<p>AskDB returns validated SQL. Run it through your own application, approval flow, or database tooling outside AskDB.</p>


```
pnpm exec askdb ask \
  --schema my-app.schema \
  --question "List the newest users"
```



## Use the Packages

<p>For application code, install the small set of packages that match the surface you need. <code>{"@askdb/core"}</code> is dialect-agnostic. Postgres-specific behavior lives in <code>{"@askdb/postgres"}</code>. Custom enrichment UIs use <code>{"@askdb/enrich"}</code>.</p>


```
pnpm add @askdb/core @askdb/postgres
# Example model provider for live generation
pnpm add ai @ai-sdk/openai
# Optional: only for live Postgres introspection
pnpm add pg
# Optional: Prisma schema-file introspection (no live database required)
pnpm add @askdb/introspect @askdb/prisma
# Optional: only for custom Schema v2 enrichment authoring
pnpm add @askdb/enrich
# Optional: browser enrichment UI served locally
pnpm add @askdb/studio
# Optional: only when you want retrieval over schema chunks
pnpm add @askdb/rag
```
