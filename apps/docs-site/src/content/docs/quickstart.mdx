<p class="doc-eyebrow">From zero to first query</p>

<p class="doc-lede">Scaffold your config, point AskDB at your database, enrich the schema artifact, and ask your first question. The same packages drop straight into a Node service when you're ready to embed.</p>

## Prerequisites

- Node 20 or newer.
- npm, pnpm, or yarn (examples use `npx`).
- An API key from a model provider — OpenAI, Anthropic, Google, OpenRouter, or any OpenAI-compatible endpoint. See [Bring your own model](/guides/bring-your-own-model/).
- A database AskDB can read — Postgres, MySQL, SQLite, or SQL Server — or a Prisma schema file (no live database required).

## 1. Scaffold your config

```bash
npx askdb@latest init
```

`npx` downloads and runs `askdb` on the fly — no separate install step needed. `askdb init` creates `askdb.config.ts` in your project root. The config file is TypeScript-checked and reads values from your environment using the `env()` helper — it loads from a `.env` file at your project root or from the system environment, whichever is set. You choose your own variable names and pass them to `env()`:

```ts
apiKey: env("OPENAI_API_KEY"),   // or env("MY_KEY") — you pick the name
```

`askdb.config.ts` is checked in; `.env` is not.

Open `askdb.config.ts` and fill in your settings:

```ts
import { defineConfig } from "@askdb/config";

export default defineConfig({
  ai: {
    provider: "openai",   // or "anthropic", "google", etc.
    model: "gpt-4o-mini",
  },
  introspection: {
    outputDir: "./my-app.schema",  // optional — defaults to ./askdb/
  },
});
```

Once this is configured, most CLI flags below are optional.

## 2. Introspect your database

Introspection produces an **AskDB schema artifact** — a directory on disk that captures your tables, columns, types, and relationships. That artifact is the only thing the model ever sees about your database.

### From a live database

With `DATABASE_URL` in `.env` and your database connection configured in `askdb.config.ts`, one command is enough. `outputDir` is optional and defaults to `./askdb/` — see the [Configuration reference](/reference/config/) for all options, their defaults, and which are required:

```bash
npx askdb introspect
```

Pass `--url` to target a different database for this run:

```bash
npx askdb introspect --url "$OTHER_DATABASE_URL"
```

AskDB picks the right dialect from the connection string. See [Switch engines](/guides/switch-engines/) for the install matrix.

### From a Prisma schema

If you already have a Prisma schema file, introspect directly from it — no live database required:

```bash
npx askdb introspect --engine prisma --prisma-schema ./prisma/schema.prisma
```

You can also set this in `askdb.config.ts` so no flags are needed:

```ts
introspection: {
  provider: "prisma",
  providerConfig: {
    prisma: { schemaPath: "./prisma/schema.prisma" },
  },
},
```

## 3. Enrich and test in Studio

Raw introspection captures structure. Enrichment adds descriptions, aliases, business concepts, sensitive markers, and example questions — the layer that makes generation good.

Launch **Studio** — the recommended authoring surface — in a browser:

```bash
npx askdb studio
```

Studio is also the best place to **test your questions**: ask a question in the same session, review the generated SQL, and keep enriching if the answer isn't what you expected — all without switching tools.

Prefer the terminal? The TUI covers the same workflow:

```bash
npx askdb enrich
```

Both commands read `outputDir` from `askdb.config.ts`. The full walkthrough lives in [Author your schema](/guides/author-your-schema/).

## 4. Ask from the CLI

Once the schema is enriched, you can also ask directly from the terminal:

```bash
npx askdb ask --question "Which customers signed up last week?"
```

AskDB returns validated SQL. It does **not** run it against your database — your application owns execution.

For iterative work — asking questions and continuing to enrich in the same session — Studio (step 3) is the better surface.
