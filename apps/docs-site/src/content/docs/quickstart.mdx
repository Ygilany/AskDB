<p className="doc-eyebrow">From install to first query</p>

<p className="doc-lede">Install AskDB, generate or point at an <strong>AskDB schema</strong> folder, optionally enrich it, and run questions from the CLI. The same packages then drop into your Node services.</p>

## Prerequisites

<ul>
            <li>Node 20 or newer.</li>
            <li>npm, pnpm, or yarn (examples use <code>{"npx"}</code> / <code>{"npm exec"}</code>).</li>
            <li>An API key for a supported AI provider — OpenAI (or any OpenAI-compatible endpoint) or Azure OpenAI — for live NL-to-SQL generation.</li>
            <li>A PostgreSQL connection string only if you want live Postgres introspection. A Prisma schema file is enough if you use <code>{"--engine prisma"}</code>.</li>
          </ul>

## Option A — Try with a sample (AskDB repo contributors)

<p>If you are working inside a clone of the AskDB repository, you can use the bundled sample schema under <code>{"fixtures/schemas/orders-users.schema"}</code> after installing dependencies and building the workspace.</p>


```
pnpm install
cp .env.example .env
# edit .env for API keys; optional: pnpm exec askdb init
pnpm build

pnpm exec askdb ask \
  --schema fixtures/schemas/orders-users.schema \
  --question "How many orders are there?"
```



## Option B — npm install (typical)

<p>Install the CLI globally or per project, add keys to <code>{".env"}</code>, then introspect your own database (or Prisma file) to produce a schema folder.</p>


```
npm install @askdb/cli
npx askdb init
# follow comments in askdb.config.ts / .env for API keys

npx askdb introspect \
  --url "$DATABASE_URL" \
  --out my-app.schema \
  --schema-id my-app

npx askdb enrich --schema my-app.schema

npx askdb ask \
  --schema my-app.schema \
  --question "Which tables are connected to users?"
```



## Create a schema from a database

### From Postgres

<p>Point introspection at a live database; AskDB writes an AskDB schema directory you can enrich and query.</p>


```
npx askdb introspect \
  --url "$DATABASE_URL" \
  --out my-app.schema \
  --schema-id my-app

npx askdb enrich --schema my-app.schema

npx askdb ask \
  --schema my-app.schema \
  --question "Which tables are connected to users?"
```



### From a Prisma schema

<p>If you already have a Prisma schema file, introspect directly from it — no live database required.</p>


```
npx askdb introspect \
  --engine prisma \
  --prisma-schema ./prisma/schema.prisma \
  --out my-app.schema \
  --schema-id my-app

npx askdb ask \
  --schema my-app.schema \
  --question "Which tables are connected to users?"
```


<p>Set <code>{"ASKDB_PRISMA_SCHEMA"}</code> in <code>{".env"}</code> to make <code>{"--prisma-schema"}</code> optional on repeat runs. See <a href="../prisma/">@askdb/prisma</a> for supported providers.</p>



## Enrich the schema

<p>Raw introspection captures structure from the catalog (or Prisma file). Enrichment adds the <strong>markdown table layer</strong>: descriptions, aliases, column notes, common query language, example questions, and concepts. AskDB ships two UIs — pick whichever fits your workflow.</p>

**Terminal UI** — keyboard-driven, works in any terminal:

```
npx askdb enrich --schema my-app.schema
```

**Studio** — browser-based UI served locally at <code>http://127.0.0.1:5556</code>:

```
npx askdb studio --schema my-app.schema
```

<p>Both UIs use <code>{"@askdb/enrich"}</code> under the hood and write the same files on disk. When you want one portable file, bundle the directory:</p>

```
# produce a single read-only artifact for distribution
npx askdb bundle my-app.schema --out my-app.schema.bundle.json
```



## SQL output only

<p>AskDB returns validated SQL. Run it through your own application, approval flow, or database tooling outside AskDB.</p>


```
npx askdb ask \
  --schema my-app.schema \
  --question "List the newest users"
```



## Use the packages

<p>For application code, install the packages that match the surface you need. <code>{"@askdb/core"}</code> is dialect-agnostic. Postgres-specific behavior lives in <code>{"@askdb/postgres"}</code>. Custom enrichment UIs use <code>{"@askdb/enrich"}</code>.</p>


```
npm add @askdb/core @askdb/postgres
# Example model provider for live generation
npm add ai @ai-sdk/openai
# Optional: only for live Postgres introspection
npm add pg
# Optional: Prisma schema-file introspection (no live database required)
npm add @askdb/introspect @askdb/prisma
# Optional: only for custom enrichment authoring in your own UI
npm add @askdb/enrich
# Optional: browser enrichment UI served locally
npm add @askdb/studio
# Optional: only when you want retrieval over schema chunks
npm add @askdb/rag
```
