<p className="doc-eyebrow">From install to first query</p>

<p className="doc-lede">Install AskDB, generate or point at an <strong>AskDB schema</strong> folder, optionally enrich it, and run questions from the CLI. The same packages then drop into your Node services.</p>

## Prerequisites

- Node 20 or newer.
- npm, pnpm, or yarn (examples use <code>{"npx"}</code> / <code>{"npm exec"}</code>).
- An API key for a supported AI provider — OpenAI (or any OpenAI-compatible endpoint) or Azure OpenAI — for live NL-to-SQL generation.
- A PostgreSQL connection string only if you want live Postgres introspection. A Prisma schema file is enough if you use <code>{"--engine prisma"}</code>.

## Typical install

Use this path when you are adding AskDB to your own project. Install the CLI globally or per project, add keys to <code>{".env"}</code>, then introspect your own database (or Prisma file) to produce a schema folder.

```
npm install askdb
npx askdb init
# follow comments in askdb.config.ts / .env for API keys

npx askdb introspect \
  --url "$DATABASE_URL" \
  --out my-app.schema \
  --schema-id my-app

npx askdb enrich --schema my-app.schema

npx askdb ask \
  --schema my-app.schema \
  --question "Which tables are connected to users?"
```

## Working from the AskDB repository

Use this path when you are contributing to AskDB itself. You can use the bundled sample schema under <code>{"fixtures/schemas/orders-users.schema"}</code> after installing dependencies and building the workspace.

```
pnpm install
cp .env.example .env
# edit .env for API keys; optional: pnpm exec askdb init
pnpm build

pnpm exec askdb ask \
  --schema fixtures/schemas/orders-users.schema \
  --question "How many orders are there?"
```

## Create a schema from a database

### From Postgres

Point introspection at a live database; AskDB writes an AskDB schema directory you can enrich and query.

```
npx askdb introspect \
  --url "$DATABASE_URL" \
  --out my-app.schema \
  --schema-id my-app

npx askdb enrich --schema my-app.schema

npx askdb ask \
  --schema my-app.schema \
  --question "Which tables are connected to users?"
```

### From a Prisma schema

If you already have a Prisma schema file, introspect directly from it — no live database required.

```
npx askdb introspect \
  --engine prisma \
  --prisma-schema ./prisma/schema.prisma \
  --out my-app.schema \
  --schema-id my-app

npx askdb ask \
  --schema my-app.schema \
  --question "Which tables are connected to users?"
```

Set <code>{"ASKDB_PRISMA_SCHEMA"}</code> in <code>{".env"}</code> to make <code>{"--prisma-schema"}</code> optional on repeat runs. See <a href="../prisma/">@askdb/prisma</a> for supported providers.

## Enrich the schema

Raw introspection captures structure from the catalog (or Prisma file). Enrichment adds the **markdown table layer**: descriptions, aliases, column notes, common query language, example questions, and concepts. AskDB ships two UIs — pick whichever fits your workflow.

**Terminal UI** — keyboard-driven, works in any terminal:

```
npx askdb enrich --schema my-app.schema
```

**Studio** — browser-based UI served locally at <code>http://127.0.0.1:5556</code>:

```
npx askdb studio --schema my-app.schema
```

Both UIs use <code>{"@askdb/enrich"}</code> under the hood and write the same files on disk. When you want one portable file, bundle the directory:

```
# produce a single read-only artifact for distribution
npx askdb bundle my-app.schema --out my-app.schema.bundle.json
```

## SQL output only

AskDB returns validated SQL. Run it through your own application, approval flow, or database tooling outside AskDB.

```
npx askdb ask \
  --schema my-app.schema \
  --question "List the newest users"
```

## Use the packages

For application code, install the packages that match the surface you need. <code>{"@askdb/core"}</code> is dialect-agnostic. Postgres-specific behavior lives in <code>{"@askdb/postgres"}</code>. Custom enrichment UIs use <code>{"@askdb/enrich"}</code>.

```
npm add @askdb/core @askdb/postgres
# Example model provider for live generation
npm add ai @ai-sdk/openai
# Optional: only for live Postgres introspection
npm add pg
# Optional: Prisma schema-file introspection (no live database required)
npm add @askdb/introspect @askdb/prisma
# Optional: only for custom enrichment authoring in your own UI
npm add @askdb/enrich
# Optional: browser enrichment UI served locally
npm add @askdb/studio
# Optional: only when you want retrieval over schema chunks
npm add @askdb/rag
```
