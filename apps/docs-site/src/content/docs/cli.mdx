<p className="doc-eyebrow">Terminal workflow</p>

<p className="doc-lede">The CLI ships the `askdb` binary for asking questions, generating SQL, logging, introspection, enrichment, and bundling.</p>

## Install



```
pnpm add -g @askdb/cli
# or run from a workspace clone
pnpm exec askdb --help
```


          <p>The CLI is the batteries-included workflow for SQL generation and schema introspection. It uses <code>{"pg"}</code> only for live Postgres introspection.</p>

## Init



```
pnpm exec askdb init
# writes ./askdb.config.ts (refuses to overwrite unless --force)
```



          <p>Use <code>{"--path"}</code> to write the template to a different file. <code>{"askdb init"}</code> does not create a <code>{".env"}</code> file; example <code>{".env"}</code> keys and notes live in comments inside <code>{"askdb.config.ts"}</code>. <code>{"askdb init"}</code> intentionally skips loading config so scaffolding cannot be blocked by a broken config file.</p>

## Ask



```
askdb ask \
  --schema ./my-app.schema \
  --question "Top 5 customers by lifetime value"
```



## Introspect



```
askdb introspect \
  --url "$DATABASE_URL" \
  --out ./my-app.schema \
  --schema-id my-app

# If ASKDB_INTROSPECT_OUT is set (e.g. from askdb.config introspection.outputDir),
# you can omit --out when you are not using --print or --diff.
```


          <p>The retired standalone <code>{"askdb-introspect"}</code> binary is no longer the documented path.</p>

## Enrich and Bundle

<p>When <code>{"@askdb/tui"}</code> is installed, the main CLI exposes thin shims for interactive enrichment and bundle creation.</p>


```
askdb enrich --schema ./my-app.schema
askdb bundle ./my-app.schema --out ./my-app.schema.bundle.json
```



## Useful Flags

<div className="definition-list">
            <div><code>{"--schema <path>"}</code><span>Path to a Schema v2 directory, bundle, or schema JSON.</span></div>
            <div><code>{"--mode <id>"}</code><span>Select <code>{"schema_only"}</code> or <code>{"bounded_results"}</code>.</span></div>
            <div><code>{"--explain"}</code><span>Print heuristic guardrail metadata.</span></div>
            <div><code>{"--log-file <path>"}</code><span>Append structured JSON logs to a file.</span></div>
          </div>
