<p className="doc-eyebrow">Reference, not the starting point</p>

<p className="doc-lede">Use <a href="../journeys/">Journeys</a> when you want a task-shaped path. This page is the <strong>package catalog</strong>: what each <code>{"@askdb/*"}</code> module owns and when it belongs in your dependency graph.</p>

## Core runtime

| Package | Use it when |
| --- | --- |
| [`@askdb/config`](../environment/) | You want Prisma-style `askdb.config.*` discovery, `defineConfig`, `env()` helpers, and `bootstrapAskDbEnv()`. |
| [`@askdb/core`](../core/) | You want schema loading, prompt assembly, SQL generation, and validation. |
| [`@askdb/postgres`](../postgres/) | You need Postgres SQL generation, validation, or catalog introspection. |
| [`@askdb/mysql`](../engines/#mysql) | You need MySQL or MariaDB SQL generation or live introspection. |
| [`@askdb/sqlite`](../engines/#sqlite) | You need SQLite SQL generation or live introspection. |
| [`@askdb/sqlserver`](../engines/#sql-server) | You need SQL Server SQL generation or live introspection. |

## Schema authoring

| Package | Use it when |
| --- | --- |
| [`@askdb/introspect`](../introspect/) | You want to create an AskDB schema artifact from a database or Prisma file. |
| [`@askdb/prisma`](../prisma/) | You want to introspect from a Prisma schema file without connecting to a database. |
| [`@askdb/enrich`](../enrich/) | You are building or extending a schema authoring workflow. |
| [`@askdb/tui`](../tui/) | You want terminal-based schema enrichment. |
| [`@askdb/studio`](../studio/) | You want browser-based schema enrichment. |

## Retrieval

| Package | Use it when |
| --- | --- |
| [`@askdb/rag`](../rag/) | Your schema is too large to send in every prompt and you need focused schema retrieval. |

## Surfaces

| Package | Use it when |
| --- | --- |
| [`askdb`](../cli/) | You want the CLI for asking questions and running introspection. |
| [`@askdb/http-api`](../http-api/) | You want to expose AskDB through HTTP behind your own auth and policy layer. |

## Monorepo development

<p>If you work inside the AskDB source repository, reusable libraries live under <code>{"packages/*"}</code> and first-party apps under <code>{"apps/*"}</code>. UI surfaces depend on shared headless packages instead of each other. Common workspace commands:</p>



```
pnpm build
pnpm test
pnpm lint
pnpm smoke:install
pnpm preflight
```
