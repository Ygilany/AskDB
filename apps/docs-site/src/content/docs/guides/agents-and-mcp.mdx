<p className="doc-eyebrow">Agent integrator journey</p>

<p className="doc-lede">
  AskDB is built for <strong>machine-friendly surfaces</strong> so agents can generate or validate SQL under the same contracts as human integrators. Today that means the <strong>CLI</strong>, <strong>library</strong>, and <strong>HTTP API</strong>. A dedicated <strong>Model Context Protocol (MCP)</strong> server is <strong>roadmap</strong> work—not shipped yet.
</p>

## What works today

- **CLI** — Stable flags and JSON-friendly logs for scripting and CI; run <code>{"askdb ask"}</code> with a schema path and question.
- **Library** — Embed <code>{"ask()"}</code> in your agent runtime with explicit schema and model wiring (<a href="./embed-in-app/">Embed NL→SQL</a>).
- **HTTP API** — Wrap the core for tools that speak HTTP (<a href="./http-api/">Run the HTTP API</a>).

## MCP status

- **MCP** is planned as a first-party surface that reuses the same core. Track progress on the <a href="https://github.com/Ygilany/AskDB">AskDB GitHub project</a> (roadmap in the repository).
- Until MCP ships, build your own thin adapter that calls <code>{"ask()"}</code> or the HTTP API, and enforce **policy** (modes, sensitive fields, which connection strings exist) in your host.

## Stable contracts for hosts

- <a href="https://github.com/Ygilany/AskDB/blob/main/docs/integration/reuse-core-phase-3.md">Core reuse patterns for HTTP/MCP-style wrappers (GitHub)</a> — entrypoints and semantics hosts should mirror.
- <a href="../../modes/">Modes and safety</a> — trust boundaries for model context.
