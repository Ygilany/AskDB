<p className="doc-eyebrow">Integrator journey</p>

<p className="doc-lede">
  You ship a service or internal tool that turns natural language into SQL. AskDB stays on the <strong>schema and generation</strong> side: you own credentials, approval, and execution against the database.
</p>

## When this guide fits

- You already have (or will generate) an AskDB **schema artifact** (folder or bundle).
- You want **programmatic** NL→SQL inside Node, not only the CLI.

## Trust boundary

- AskDB returns **validated read-only SQL** (PostgreSQL <code>{"SELECT"}</code> guardrails today). It does **not** run that SQL against your database.
- You choose operating <a href="../../modes/">modes</a> and how much schema context reaches the model; sensitive-field handling is documented under <a href="../../modes/">Modes and safety</a>.

## Steps

1. **Install** — <code>{"@askdb/core"}</code>, <code>{"@askdb/postgres"}</code>, and your AI SDK provider packages (for example <code>{"ai"}</code> and <code>{"@ai-sdk/openai"}</code>). Add <code>{"pg"}</code> only if you also use live Postgres introspection from this app.
2. **Configure** (optional) — Use <a href="../../environment/"><code>{"@askdb/config"}</code></a> so <code>{"bootstrapAskDbEnv()"}</code> loads <code>{"askdb.config.ts"}</code> and a runtime snapshot for model env maps.
3. **Load schema** — <code>{"loadSchema(\"./my-app.schema\")"}</code> from a directory, bundle, or <code>{"schema.json"}</code>.
4. **Call** <code>{"ask()"}</code> — Pass <code>{"postgresDialect"}</code>, the question, schema, and model. See <a href="../../core/">@askdb/core</a> for the full API and logging hooks.
5. **Execute** — Run the returned SQL only after your own checks, read-only roles, and tenant policy.

## Large schemas

If prompts get too large, add <a href="../rag-large-schema/">retrieval over your schema</a> with <code>{"@askdb/rag"}</code> instead of stuffing the whole artifact into every request.
