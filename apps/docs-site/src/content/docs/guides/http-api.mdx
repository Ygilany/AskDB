<p className="doc-eyebrow">Service journey</p>

<p className="doc-lede">
  The <code>{"@askdb/http-api"}</code> package wraps the same <code>{"@askdb/core"}</code> pipeline as the CLI. Use it when a browser or another service should call AskDB over HTTP instead of linking the library directly.
</p>

## When this guide fits

- You want a **small server** that accepts questions and returns SQL (or related JSON) for a known schema deployment.
- You will put **authentication, rate limits, and allowlists** in front of the API in your own stack.

## Trust boundary

- Treat the HTTP surface like any internal tool: **no anonymous public internet** exposure without your own auth.
- AskDB still returns **SQL only** on current surfaces; execution stays **host-owned**.

## Steps

1. **Run from the monorepo** during development or depend on the published package in your service (see <a href="../../http-api/">@askdb/http-api</a> for flags and env).
2. **Bootstrap config** — Align <code>{"askdb.config.ts"}</code> and <code>{"bootstrapAskDbEnv()"}</code> with how you load secrets in production (<a href="../../environment/">Environment</a>).
3. **Mount behind your gateway** — Terminate TLS and auth at your edge or API gateway.
