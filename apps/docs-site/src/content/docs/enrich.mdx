<p className="doc-eyebrow">Headless enrichment</p>

<p className="doc-lede">The enrich package owns reusable schema authoring workflow logic for TUI, Studio, and custom enrichment UIs.</p>

## When to use

<ul>
  <li>You build a <strong>custom authoring UI</strong> and want the same workspace operations as first-party surfaces.</li>
  <li>You are extending Studio/TUI behavior programmatically—see <a href="../guides/author-enrich-schema/">Author and enrich schema</a>.</li>
  <li>You need headless helpers to load, edit, save, and bundle an AskDB schema directory—use <code>{"@askdb/tui"}</code> or <code>{"@askdb/studio"}</code> for the maintained UIs.</li>
</ul>

## Install



```
npm add @askdb/enrich
# usually paired with schema primitives
npm add @askdb/core
```



## What it owns

<ul>
            <li>Loading a schema directory as an editable workspace.</li>
            <li>Building table drafts and front-matter from physical schema plus markdown.</li>
            <li>Saving <code>{"tables/*.md"}</code> and <code>{"concepts.md"}</code>.</li>
            <li>Preserving markdown body sections during scoped edits.</li>
            <li>Building AI suggestion targets and context.</li>
            <li>Bundling a split schema directory into a single JSON artifact.</li>
          </ul>

## Package boundary

<p><code>{"@askdb/enrich"}</code> depends on <code>{"@askdb/core"}</code>. Authoring surfaces such as <code>{"@askdb/tui"}</code> and <code>{"@askdb/studio"}</code> depend on <code>{"@askdb/enrich"}</code>. Studio does not depend on the TUI.</p>


```
@askdb/core
   ^
   |
@askdb/enrich
   ^          ^
   |          |
@askdb/tui   @askdb/studio
```
